# 🎓 Exam Seating Management System

A modern, full-stack university-grade platform designed to automate and optimize exam seating arrangements. This system eliminates departmental conflicts, streamlines student registration, and provides secure, time-restricted administrative controls.

---

## ✨ Key Features

### 🏢 University Management
-   **Student Command Center**: Self-service registration flow where students manage their own profiles (Roll No, Enrollment, Department, etc.).
-   **Admin Hub**: Comprehensive CRUD operations for university Rooms, Exams, and Student records.
-   **Dynamic Room Configuration**: Define room capacities, bench counts, and physical locations (Blocks/Floors).

### 🧠 Smart Seating Engine
-   **Anti-Conflict Algorithm**: Automatically interleaves students from different departments cyclically to ensure no two students from the same department sit adjacent.
-   **Shift & Timing Awareness**: Handles Morning, Afternoon, and Evening shifts with specific exam types (Regular, Backlog, etc.).

### 🔒 Security & Compliance
-   **Time-Locked PDF Generation**: Seating plans can be exported to industry-standard PDF format using Puppeteer.
-   **Download Restriction**: Final seating plans are securely locked until **at least 2 hours before the exam start time** to prevent unauthorized early access.
-   **JWT Authentication**: Secure, role-based access control for Admins and Students.

### 🍱 Premium Aesthetics
-   **Modern Glassmorphism**: A sleek, responsive user interface built with modern design principles.
-   **Dark/Light Mode**: Full support for system-based or manual theme switching.
-   **Micro-animations**: Smooth transitions powered by Framer Motion.

---

## 🚀 Tech Stack

-   **Frontend**: React 19, Vite, Tailwind CSS, Framer Motion, Lucide Icons.
-   **Backend**: Node.js, Express.js.
-   **Database**: MySQL (Managed via `mysql2`).
-   **Export Service**: Puppeteer (Headless PDF generation).
-   **State Management**: Axios Interceptors for token-based authentication.

## 🛠️ Installation & Setup

### Prerequisites
-   [Node.js](https://nodejs.org/) (v18+)
-   [MySQL Server](https://www.mysql.com/)

### 1. Clone the Repository
```bash
git clone https://github.com/Arjunsingh-ops/exam-system.git
cd exam-system
```

### 2. Backend Configuration
1. Navigate to the backend directory: `cd backend`
2. Install dependencies: `npm install`
3. Create a `.env` file based on the environment variables needed:
   ```env
   PORT=5000
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_password
   DB_NAME=exam_seating_system
   JWT_SECRET=your_super_secret_key
   ```
4. Start the backend: `npm run dev` (The system will automatically initialize the database schema on first run).

### 3. Frontend Configuration
1. Navigate back to the root: `cd ..`
2. Install dependencies: `npm install`
3. Start the development server: `npm run dev`

## 📖 Usage Guide

### For Administrators
1. **Login**: Use the default credentials (Email: `admin@exam.edu`, Password: `password`).
2. **Setup Rooms**: Navigate to the **Rooms** section to add your university's examination halls.
3. **Manage Exams**: Add exams with specific dates, shifts (Morning/Evening), and types.
4. **Generate Seating**: Go to **Seating Plan**, select the Target Exam and the available Rooms, then click **Generate**.
5. **Download PDF**: The PDF option becomes available only 2 hours before the exam begins.

### For Students
1. **Register**: Sign up at `/register`.
2. **Update Profile**: Log in and fill out your enrollment details in the **Profile** section.
3. **View Seating**: Your seating arrangement will appear on your dashboard once generated by the admin.
